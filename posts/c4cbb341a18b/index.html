一、结构体基本声明：

结构体的内存大小是其中所有结构变量的大小之和，有时可能会大一点，因为有的系统可能会有对齐机制，所以会有缝隙。

1.一般声明方法：

  1. tag : 标签（非必须）
  2. … 结构体成员
  3. variable——lsit：结构体变量

同时也可以使用typedef来方便的声明变量；

2.便捷的声明方法：

使用第一种声明时，若生成新的结构体变量要：
struct tag variable_Name;
第二种：
simple Variable_Name;

奇怪是是结构体这里如果应用一个指针反而会语法错误，只需要引用变量名称；不知道这是否可以佐证*符号应该和类型一起看，而不是和变量名一起看。解决一个c语言千古难题，哈哈

结构体成员可以包含其他结构体，也可以包含指向自己结构体的类型的指针。-实现链表和树。

二、共用体（union）声明：

相同的内存位置存储不同的数据类型；共用体占用内存的位置是存储中的最大的成员。

共用体我能想的的用法就是做临时变量，但这个栈中的临时变量也能完成，或许共用体可以方便其他函数能够调用到。

共用体同一时间只能有一个数据有效。

三、位域（c_bit_field):

带有预定义宽度的变量。
In C, we can specify the size (in bits) of the structure and union members. The idea of bit-field is to use memory efficiently when we know that the value of a field or group of fields will never exceed a limit or is within a small range. Bit fields are used when the storage of our program is limited. Need of bit fields in C programming language:

  • Reduces memory consumption.

  • To make our program more efficient and flexible.

  • Easy to Implement.

    struct
    {
    data_type member_name: width_of_bit-field;
    };

一个位域声明至少有4Byte**（32位），最多支持32个只需一bit的内存的变量。若超过一位如33个1bit则再开辟4 Byte内存。**

元素 描述
data_type 它是一种整数类型，用于确定要解释的位字段值。类型可以是 int、有符号 int 或无符号 int。
member_name 成员名称是位字段的名称，即变量名
宽度 位字段中的位数。宽度必须小于或等于指定类型的位宽

这里需要注意有符号int和无符号int的实际取值范围其实不一。
以4位为例子，那么可以16个数字(2^4)；
若是int，那么数值是-87（中间有0）
若是unsigned int，那么数值是015
